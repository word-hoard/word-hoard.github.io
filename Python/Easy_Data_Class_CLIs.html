Easy Data Class CLIs

Even or perhaps especially for small projects, being able to add a lightweight command line interface is handy if you can do it quickly. The brilliant rich library is transformative for pretty, low effort representations of your classes from rich import print just makes life better, but a sprinkling of functions for default styles lifts this up anther notch.

from os import system, name as os_name

from rich import print

from rich.padding import Padding
from rich.rule import Rule

def para(arg):
    print(Padding(arg, (0, 5)))

def rule(arg):
    print(Rule(arg))

def clear():
    if os_name == "posix":
        system("clear")
    else:
        system("cls")

Huh...

cmd does this... but a cmd interface class gets its __init__ from inheritance, and I want to use it to interact with a data class, which gets its __init__ from a decorator. So do I have to write a sperate interface class for every data class?

So it turns out the answer is no you can make a reasonably ok interface generator, but at the cost of compromise and loosing some of cmd's superpowers. Obviously I'd like it to do better but like all craft 'this will do for now.'

This kind of highlights a personal fixation of mine with keeping __init__ simple and putting any elaborate data conversion shenanigans in a class or static method that generates an instance of the class. So I only call Myclass('exactly the arguments that I want') with safe, often hard coded data, and anywhere else I use Myclass.from_specified_data_source("dodgy_data.csv"). This allows for multiple entry points into a class that don't fight inheritance battles with each other. It also allows me to use data classes almost everywhere, which sounds daft but ends up with nicely predictable code.

The Command Line Interface class

The interface class will search for any methods in the data class starting with do_ on the class it gets composited into and repurposes them as cli specific methods in the normal CMD way. This sadly looses the really nice cmd help descriptors because the do_ commands are created after initalization, but we can compensate by adding a menu of commands that lean heavily on method doc strings and by making the menu more readable with rich. 

from cmd import Cmd

class Interface(Cmd):

    prompt = "::> "

    def do_q(self, arg):
        "quit the menu"
        self.postcmd(stop= True, line="end")
        return True

    @staticmethod
    def from_dataclass(obj: object) -> None:
        cli = Interface()
        func_names = [name for name in dir(obj)
                if name.startswith("do_")]
        cli.create_menu(obj, func_names)
        def print_menu(stop=False, line="end"):
            if stop:
                return True
            para(cli.menu)

        setattr(cli, 'preloop', print_menu)
        setattr(cli, 'postcmd', print_menu)
        for fn in func_names:
            setattr(cli, fn, getattr(obj, fn))
        cli.cmdloop()

    def create_menu(self, obj, func_names: list):
        menu_str = "[bold #3c1bc2]COMMANDS[/bold #3c1bc2]\n"
        for name in func_names:
            command = name.split("_")[1]
            fn = getattr(obj, name)
            menu_str += f"[bold #6851c2]{command}[/bold #6851c2] [i] {fn.__doc__} [/i]\n"
        self.menu = menu_str

Hurrah! We now have all our frankly ugly boilerplate corralled in one place, and our data class can use it whilst still looking clean and readable

@dataclass
class Test:
    """ sample test class for Interface class use"""

    val: str

    def status(self):
        'prints a rich formatted summary of the class'
        clear()
        rule(self.__doc__)
        para(f'val: {self.val}')
        rule("")

    def interface(self):
        'launches an interactive cli for the class'
        self.status()
        Interface.from_dataclass(self)

    def do_1(self, arg):
        'echo back the val and arg'
        self.status()
        para(f'val: [i yellow]{self.val} [/i yellow] arg: [bold]{arg}[/bold]')

    def do_setval(self, arg):
        'change my value'
        self.val = arg
        self.status()
