3 Iteration

What is a Iteration?

Iteration is sort of the whole point of using a computer. Sure you can get computers to do all sorts of fancy other stuff, but iteration is the power of not just doing the right thing once, but doing it billions of times. of course you have to be careful not to do the wrong thing billions of times....

1 Iterating through a task where you know all the data ahead of time

This is by far the most common sort of loop that comes up in python. Say you want to send an e-mail to every person in a group automatically once a month. Your write out the code for sending one email, and then use it for every person. The Python List is often the best tool for the job.

Recap: Lists

A list in python is a data type:

group_members = ["Bill", "Ted", "Chani", "Leah"]

The Rules for lists:

  • they are written with [] square brackets
  • they can have any number of data chunks inside them, including empty lists with no data
  • each piece of data must be separated by a comma
  • the data can be of different types, but it is usually more useful if they are all the same...

group_members = ["Bill", "Ted", "Chani", "Leah"]

answers = []

for person in group_members:
    answer = input(f"Hello {person} how are you?")

If you just need to do a task a set number of times, python can even create the list of numbers you need. Try this:

for i in range(4):
    print(f"{i=} haha! I love to count!")

Notice that in code you start with a zero and stop before you get to the final number. Whilst this seems crazy it is really useful, but don't worry about it for the moment, you still get four numbers.

2 Iterating when you might end at any time.

Sometimes you don't know ahead of time how long to repeat a loop for. Think of a task like looking for lost keys. You only need to search for them until you find them, then you stop. This is a very common task in games, you play a level until you have won it, a monster fights until it has no more health, etc.

In order to do this you need to understand HOW TO TEST THE TRUTH so click on the link and learn that first.

We can tell python to do a task until one (or more) things change from false to true or true to false. We call this a while loop and it works like this:

Exercise 2

This code will loop until you guess bob's name, type it out and run it a few times to get a feel for what it does.

name = ""
while name != 'bob':
    name =  input("what is my name? ")

print("you guessed my  name!")

There are risks with using while loops - if you code them badly you can get stuck in them forever - this is the reason that games are some of the worst software for crashing, because almost every game element has one or more loops running at once.

If this happens with your own can press the stop button in mu or type Ctrl+C to break out of your loop. Or at worst close the app that you are running python on. Don't worry, it happens to the best coders!

Gold task - fixing common problems

Often we get the right data in the wrong format or in a messy way. So with "2" != 2 a string with the same number in it as an integer does not match so Python will call this True.

The input function returns (or gives back) any user input as a string. Which means that if you try matching it with an integer it won't work. However you can convert the string to an integer like this:

guess = 0
while guess != 3:
    guess =  input("what is my number? ")
    guess = int(guess)

int(guess) converts the guess to an integer - well it tries to but you can't convert All to 411, str(guess) converts integers to strings fairly reliably though.
