1 Variables

What is a Variable?

A variable is a labeled piece of data. What does that mean? Its dead simple! Think about the sentence "My name is Pink." the piece of data is the word "Pink" and the label is "name". In python we would write it like this:

name = "Pink"

and we can get python to print it out like this:

print(name)

Why do we need variables?

We need variables if we want to store and reuse that data. We don't want to have to ask for someone's name every time we want to use in a sentence. And if you are like me you might envy a computers ability to remember a name if you only say it once.

what should we call our variables?

Anything we like! Except we can't use words python already uses - print for example, so never write print = "something", but you can write print_this or to_be_printedsafely. Notice how I sneaked in another rule - you can't have spaces in your variable names, so use and underscore_` instead. Why? well because you can assign more than one variable at a time like this:

name, job = "Pink", "Singer"

Be careful when you do this though, as keeping your code clear and readable makes it much more likely that you can spot mistakes.

what can we do with variables?

Lots of things! We might need to join them together, like this:

person_details = name + " is a " + job
print(person_details)

# prints out "Pink is a singer"

That's ok, but there is a better way for inserting your variables into a chunk of text call f-strings because they start with an f. They work like this:

customer = "Njala Omi"
favorite_food = "cheese cake"

greeting = f"hello {customer} would you like to order {favorite_food}?"

This is a bit easier to read, and can be reused as a template for greeting other customers with only a little bit of extra code.

  • print out each of your variables
  • print out an f-string using some of your variable.
